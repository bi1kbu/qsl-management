# 场景拆分重构冻结方案（V1）

更新时间：2026-05-02
状态：已按当前代码事实同步（作为后续开发与验收对齐基线）

## 1. 总目标

1. 将现有菜单重构为四个一级菜单：`通联业务`、`线上换卡业务`、`线下换卡业务`、`收卡业务`。
2. 每个一级菜单内形成完整可追踪的卡片发出流。
3. 同步补齐前台自助页面与前台卡片能力。

## 2. 业务场景主字段（冻结）

字段名：`sceneType`
可选值（固定）：

1. `QSO`
2. `SWL`
3. `ONLINE_EYEBALL`
4. `EYEBALL`

说明：

1. `QSO` 与 `SWL` 为通联业务场景。
2. `ONLINE_EYEBALL` 为线上换卡场景。
3. `EYEBALL` 为线下换卡场景。

## 3. 菜单重构（冻结）

### 3.1 一级菜单

1. 通联业务
2. 线上换卡业务
3. 线下换卡业务
4. 收卡业务

### 3.2 二级菜单

#### 通联业务

1. 通联日志
2. 创建卡片
3. 制卡签发
4. 发信确认
5. 送达确认

#### 线上换卡业务

1. 换卡申请审核
2. 创建卡片
3. 制卡签发
4. 发信确认
5. 送达确认

#### 线下换卡业务

1. 创建活动
2. 创建卡片
3. 制卡签发
4. 送达确认

#### 收卡业务

1. 通联收卡
2. 线上换卡收卡
3. 线下换卡收卡

## 4. 命名替换（冻结）

1. 现有“收信确认”统一更名为“送达确认”。
2. 页面标题、菜单名、按钮文案、文档、接口说明统一使用“送达确认”。

## 5. 流程与能力约束（冻结）

### 5.1 通联业务

1. 通联日志默认 `QSO`。
2. 可通过下拉标记为 `SWL`。
3. `QSO` 与 `SWL` 区别仅体现在通联日志字段差异。
4. 创建卡片负责关联：`卡片编号`、`日志条目`、`收件地址`。

### 5.2 线上换卡业务

1. 前台申请 -> 后台审核。
2. 审核通过后自动创建卡片。
3. 后续进入制卡签发、发信确认、送达确认流程。

### 5.3 线下换卡业务

1. 支持创建活动（名称、地点、日期时间、备注等）。
2. 支持创建“待填写呼号”的活动卡片。
3. 送达确认阶段允许补录呼号并触发发卡状态变化（已确认允许）。

### 5.4 收卡业务

1. 三类收卡业务全部支持批量能力（已确认）。
2. 收卡编号与卡片ID关联，支持一份卡片ID对应多个收卡编号。

## 6. 前台页面与卡片（冻结）

需提供对应前台页面与前台卡片（可用于相关文章/页面插入）：

1. 线上换卡场景
2. 线下换卡场景
3. 送达确认场景（原收信确认）

## 7. 本次确认项（用户最终确认）

1. 业务场景主字段使用：`QSO`、`SWL`、`ONLINE_EYEBALL`、`EYEBALL`。
2. 同意将“收信确认”替换为“送达确认”。
3. 三类收卡业务都需要批量能力。
4. 线下换卡业务送达确认允许补录呼号并触发状态变化。

## 9. 落地进度（2026-04-30）

### 9.1 已完成

1. 菜单骨架已拆分为四个一级菜单，并按冻结方案挂载二级菜单。
2. `CardRecord` 持久化模型已增加 `sceneType` 字段。
3. `创建卡片`、`制卡签发`、`发信确认`、`送达确认` 已支持按 `sceneType` 过滤数据。
4. `送达确认` 接口已扩展 `sceneType` 入参，避免同呼号同卡片类型跨场景串改。
5. 线下换卡业务“创建活动”已落地为可持久化页面（新增/编辑/删除）。
6. 线下换卡业务“创建卡片”已支持关联活动并持久化到卡片记录。

### 9.2 当前代码事实

1. 线下活动已持久化为 `OfflineActivity`，页面模块为 `OfflineActivityModule.vue`。
2. 线下换卡卡片通过 `CardRecord.spec.offlineActivityName` 关联活动。
3. “送达确认”接口入参已包含 `sceneType`，服务端匹配卡片时会同时校验 `callSign`、`cardType` 与 `sceneType`。
4. 收卡编号已落地为 `CardRecord.spec.receivedRecordCodes`，支持同一卡片记录累计多个收卡编号。
5. “卡片异动”菜单已落地在数据菜单中，权限节点为 `card-mutation`。

### 9.3 本轮新增完成（2026-04-30）

1. 新增前台线上换卡申请页：`/apis/api.qsl-management.halo.run/v1alpha1/ONLINE_EYEBALL` 与 `/ONLINE_EYEBALL/{cardId}`。
2. 新增前台线下换卡确认页：`/apis/api.qsl-management.halo.run/v1alpha1/EYEBALL` 与 `/EYEBALL/{cardId}`。
3. 新增文章/页面短码：`[qsl-online-exchange-card]`、`[qsl-offline-exchange-card]`，分别插入线上/线下换卡卡片。
4. 公开查询页与签收页支持 `sceneType` 透传，公开查询接口与公开签收接口支持按 `sceneType` 过滤与校验。
5. 公开换卡申请数据已持久化 `sceneType`（`ExchangeRequest.spec.sceneType`）。
6. `通联日志` 已支持 `QSO/SWL` 场景切换与字段差异化渲染（同页下拉切换，SWL 自动收敛非必要字段）。
