# QSL 管理系统数据字段与组织结构汇总

> 核验日期：2026-04-21  
> 核验范围：`qsl-plugin/src/main/java/com/bi1kbu/qslmanagement/extension/model/*.java` 与资源命名实现

## 1. 存储组织总览

- 存储机制：Halo Extension（CRD 风格）
- API Group/Version：`qsl-management.halo.run/v1alpha1`
- 统一访问路径：`/apis/qsl-management.halo.run/v1alpha1/{plural}`
- 统一对象结构：
  - `apiVersion`：资源所属 API 版本标识
  - `kind`：资源类型名
  - `metadata.name`：资源唯一 ID（持久化主键）
  - `metadata.version`：乐观锁版本号（更新时用于并发控制）
  - `metadata.creationTimestamp`：创建时间
  - `spec`：业务字段主体
  - `status`：流程状态/派生状态字段

## 2. 资源与字段清单（含中文含义）

## 2.1 StationProfile（通信地址）

- kind/plural：`StationProfile` / `station-profiles`
- 典型资源名：`qsl-station-profile-default`（单例）

`spec` 字段：
- `myCallSign`：本台呼号
- `myName`：本台姓名（My_Name）
- `myTelephone`：本台联系电话
- `myPostalCode`：本台邮政编码
- `myAddress`：本台通信地址
- `myEmail`：本台邮箱
- `stationRemarks`：本台备注

`status` 字段：
- `lastModifiedBy`：最后修改人
- `lastModifiedAt`：最后修改时间

## 2.2 StationEquipment（本台设备）

- kind/plural：`StationEquipment` / `station-equipments`
- 资源形态：多条（每条代表一个设备条目）

`spec` 字段：
- `rigName`：设备名称/机型
- `antennas`：可选天线列表
- `powers`：可选功率列表
- `modes`：可选工作模式列表
- `remarks`：设备备注

`status` 字段：
- `enabled`：是否启用

## 2.3 StationCard（本台卡片）

- kind/plural：`StationCard` / `station-cards`
- 资源形态：多条（按卡片版本管理）

`spec` 字段：
- `cardVersion`：卡片版本号
- `imageUrl`：卡片图像地址
- `imageMediaType`：图像 MIME 类型
- `remarks`：卡片备注

`status` 字段：
- `active`：是否激活

## 2.4 SystemSetting（系统参数）

- kind/plural：`SystemSetting` / `system-settings`
- 典型资源名：`qsl-system-setting-default`（单例）

`spec` 字段：

- `requiresExchangeReview`：换卡申请是否需要审核

`status` 字段：
- `lastModifiedBy`：最后修改人

## 2.6 QsoRecord（通联日志）

`spec` 字段：

## 3. 数据组织关系（当前实现）

- QSO 与卡片：`CardRecord.spec.qsoRecordName` 关联 `QsoRecord.metadata.name`。
- 呼号主线：`QsoRecord.spec.callSign`、`CardRecord.spec.callSign`、`AddressBookEntry.spec.callSign`、`ExchangeRequest.spec.callSign` 使用同一业务键（呼号）。
- 地址与邮件：卡片邮件目标地址最终落在 `CardRecord.spec.mailTargetEmail`，来源于地址簿匹配及发送流程写回。
- 审核到卡片：`ExchangeRequest` 审核通过后可自动生成 `CardRecord`（含 `flowStatus` 流程状态）。
- 制卡流程：制卡签发会更新 `CardRecord.spec.cardIssued/cardIssuedAt`，并将 `CardRecord.status.flowStatus` 置为“已制卡”。
- 审计：写操作通过 `QslAuditLog` 追加记录，`resourceType + resourceName` 指向受影响对象。

## 4. 关键枚举/取值组织（当前代码约束）

- 卡片类型：`QSO` / `SWL` / `EYEBALL`（多处校验采用该集合）。
- 导入导出支持数据集：
  - `qso-record`
  - `card-record`
  - `exchange-request-review`
  - `address-management`
  - `bureau-management`
  - `equipment-catalog`
- 导入导出格式：`csv` / `zip`
- 导入策略：`skip` / `overwrite`

## 5. 资源命名规则汇总

- QSO：`QSO{n}`，`n` 从 `1001` 起递增。
- 卡片：`C{n}`，`n` 从 `1001` 起递增。
- 地址：`{CALLSIGN}-{n}`，`n` 从 `1` 起递增，`CALLSIGN` 强制大写。
- 卡片局：`BURO-{n}`，`n` 从 `1` 起递增。
- 单例：
  - 通信地址：`qsl-station-profile-default`
  - 系统参数：`qsl-system-setting-default`
