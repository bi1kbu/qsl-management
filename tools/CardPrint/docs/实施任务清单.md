# CardPrint 实施任务清单

## Phase 1：工程初始化与打印基础

### 任务

1. 初始化 `pyproject.toml`、依赖、`src` 目录。  
2. 建立 `core/models.py` 的预设与任务模型。  
3. 实现 `printer/win32_adapter.py`：
   - 枚举打印机。
   - 枚举纸张。
   - 打印简单文本（连通性验证）。
4. 搭建 `cli.py` 基础命令路由。  

### 产出物

1. 可执行命令：`cardprint printer list`。  
2. 可执行命令：`cardprint printer papers --printer xxx`。  
3. 基础模型与序列化/反序列化。  

### 验收标准

1. 命令输出为标准 JSON。  
2. 在目标 Windows 机器上可识别实际针式打印机。  

## Phase 2：标定核心能力

### 任务

1. 实现 `core/transform.py`（旋转 + 原点偏移）。  
2. 实现 `core/layout_engine.py`（字段布局、死区检测）。  
3. CLI 新增：
   - `calibrate print-cross`
   - `preset validate`
4. 预设读写服务 `preset_service.py`。  

### 产出物

1. 标定页打印能力。  
2. 可保存并校验 `preset.json`。  
3. 死区冲突检查可用。  

### 验收标准

1. 单元测试覆盖 0/90/180/270 变换。  
2. 死区内字段被拒绝保存并返回可读错误。  

## Phase 3：标定 UI

### 任务

1. 实现 `ui/calibrator_app.py`（中文界面，左参右预览）。  
2. 实现调用 CLI 的进程适配器（统一 JSON 协议）。  
3. 实现字段编辑器、死区可视化（红色半透明遮罩）。  
4. 完成“保存预设”全流程。  

### 产出物

1. 可视化标定工具。  
2. 可落地的预设文件。  

### 验收标准

1. 标定参数修改后预览即时更新。  
2. 生成的预设可被 CLI 校验通过。  

## Phase 4：打印 UI 与数据导入

### 任务

1. 实现 `ui/printer_app.py`。  
2. 预设加载后标定参数只读展示。  
3. 支持手工输入、CSV、XLSX 导入。  
4. CLI 新增：
   - `render preview`
   - `print run`
5. 打印队列与逐行结果显示。  

### 产出物

1. 正式打印工具。  
2. 批量数据导入与打印能力。  

### 验收标准

1. 同一条记录在 UI 预览与实际打印位置一致。  
2. 导入 100 行数据可稳定执行并返回逐行状态。  

## Phase 5：联调、测试、发布准备

### 任务

1. 完善 `pytest` 测试：模型、CLI、变换、死区。  
2. 新增操作文档与故障排查文档。  
3. 打包发布（建议 `pipx` 或内部安装脚本）。  
4. 预留 JSON 插件接口示例（可空实现）。  

### 产出物

1. 可回归测试集。  
2. 发布包与用户使用说明。  

### 验收标准

1. 关键测试全部通过。  
2. 新环境从安装到首张标定页打印时间 < 30 分钟。  

## 横向约束（全阶段）

1. UI 不得自行实现坐标变换，必须调用 CLI/核心引擎。  
2. 预设版本化管理：`version` 字段必填。  
3. 所有错误返回必须包含 `code/message/details`。  
4. 对用户可见文案统一中文。
