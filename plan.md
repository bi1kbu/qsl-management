# QSL 插件后续开发计划（一期收口）

更新时间：2026-04-15

## 当前进度结论

1. 阶段 0-8：已完成或基本完成。
2. 阶段 9：已具备基础构建与部分单元测试，但自动化回归覆盖仍不足。

## 已完成项（摘要）

1. 后端扩展模型、主要控制台 API、公开 API 已落地并支持持久化。
2. 菜单与页面主框架已完成，前端主要模块已接入真实 API。
3. 导入导出已支持单项 CSV / 全量 ZIP 导出。
4. 导入任务已支持错误明细持久化与错误回执下载（CSV）。
5. 公开接口已接入服务端限流（按 `endpoint + clientIp`）与输入校验防滥用。
6. 已补充阶段 9-1 测试覆盖：公开接口限流/输入校验、导入错误回执服务、IP 识别工具、控制台导入任务未认证访问拦截。
7. 已补充权限模板合同测试：自动校验 `qsl-menu-role-templates.yaml` 覆盖控制台/公开 CustomEndpoint 的 `apiGroup + resource + verb` 规则。

## 后续开发顺序（从现在开始）

1. **阶段 9-1：测试基线补齐（优先级 P0）**
   - 范围：公开接口限流、公开接口输入校验、导入导出任务、关键业务动作。
   - 产出：单元测试 + API 集成测试（成功路径/失败路径/越权路径）。
   - 完成判定：关键模块具备可重复执行的自动化回归用例。
   - 当前状态：进行中（公开接口与导入错误回执已覆盖，业务动作与更多鉴权回归待补齐）。

2. **阶段 9-2：导入链路服务端化增强（优先级 P0）**
   - 范围：将当前“前端逐条写入 + 回填任务状态”进一步收敛为“服务端执行导入任务”为主。
   - 产出：导入任务执行器、任务状态流转（待处理/处理中/已完成/部分成功/失败）、统一错误收集。
   - 完成判定：导入核心逻辑在服务端闭环，前端仅负责上传与任务查询。
   - 当前状态：进行中（已切换为服务端执行导入并返回 `success/skipped/failed` 统计，`imports/precheck` 已改为服务端统一预检并与执行共用规则；后续补充更完整的任务状态流转）。

3. **阶段 9-3：权限回归与安全加固（优先级 P1）**
   - 范围：逐接口校验 `view/edit` 规则、匿名接口 RBAC 聚合检查、错误码一致性检查。
   - 产出：权限回归清单、越权用例、修复项闭环。
   - 完成判定：越权访问稳定返回 `401/403`，匿名接口仅暴露允许能力。
   - 当前状态：进行中（已补齐控制台自定义 API 与总览接口的未认证 `401` 回归、授权不足 `403` 错误码回归，以及权限模板合同自动校验；下一步补充基于真实 RBAC 规则的运行态集成验收）。

4. **阶段 9-4：导入导出稳定性测试（优先级 P1）**
   - 范围：大文件 CSV/ZIP、异常字符、重复数据、并发导入任务。
   - 产出：压力测试记录、失败恢复策略、性能基线（耗时/错误率）。
   - 完成判定：满足一期可接受的稳定性阈值并形成结论。

5. **阶段 9-5：发布准备与验收（优先级 P0）**
   - 范围：发布说明、已知限制、回滚步骤、部署与验证清单。
   - 产出：一期发布包、验收记录、发布文档。
   - 完成判定：`./gradlew.bat build` 持续通过，验收清单全部勾选完成。

## 本周建议执行节奏

1. 第 1-2 天：完成阶段 9-1（测试补齐）。
2. 第 3-4 天：完成阶段 9-2（导入服务端化增强）。
3. 第 5 天：完成阶段 9-3 与阶段 9-5（收口发布）。

## 风险与应对

1. 风险：导入链路目前仍有前端执行路径，复杂场景一致性风险较高。  
   应对：优先推进“服务端执行导入任务”并保留错误回执能力。
2. 风险：限流当前为插件进程内计数，多实例部署下配额不一致。  
   应对：二期升级网关层或共享存储限流。
3. 风险：自动化测试覆盖不足导致回归风险。  
   应对：先覆盖高风险路径（公开接口、导入导出、审批动作）。
