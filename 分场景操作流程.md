# QSL 插件分场景操作流程

本文按实际业务场景整理操作步骤，默认插件已启用，且已完成基础配置。

## 场景1：首次初始化（管理员）

1. 进入 `QSL管理 -> 配置 -> 本站配置`，填写：
   - 本台呼号
   - 通信地址（姓名、地址、邮编、电话、邮箱）
   - 系统参数（如是否启用补卡、补卡间隔天数、查询限流）
2. 进入 `QSL管理 -> 配置 -> 卡局配置`，维护常用卡局地址。
3. 进入 `QSL管理 -> 配置 -> 设备库`，维护设备/天线/功率/模式字典。
4. 在 Halo 用户组中配置 HAM、操作员、管理员权限节点。

## 场景2：用户呼号绑定申请（HAM权限 前置）

1. 登录用户进入 `UC -> QSL管理 -> 呼号绑定`。
2. 在模块1输入呼号，点击“搜索是否产生过记录”确认历史。
3. 在模块2提交三选一证明材料之一：
   - 电台执照图片
   - HamCQ 认证截图
   - 既往卡片ID + 手机号
4. 点击“提交绑定申请”。

## 场景3：呼号绑定审核（管理员/审核员）

1. 进入 `QSL管理 -> 审核 -> 呼号绑定审核`。
2. 查看申请材料，执行：
   - 通过：申请状态改为通过；按本站配置中的“审核通过赋权角色”给用户加角色。
   - 拒绝：填写拒绝原因并提交。
3. 通过后，HAM用户可在 UC 侧访问呼号管理能力（地址维护、记录查询）。

## 场景4：前台换卡申请（公开页面/页面卡片）

1. 在页面中插入“换卡申请”卡片，卡片路由：`/plugins/qsl-management/widgets/exchange`。
2. 访客填写：呼号、姓名（可选）、邮编、地址、电话（可选）、邮箱。
3. 提交后系统生成换卡申请记录（眼球卡流程）。

## 场景5：后台审核换卡申请

1. 进入 `QSL管理 -> 审核 -> 换卡申请`。
2. 审核通过：
   - 自动生成或关联卡片记录（待发状态）。
   - 发送审核结果邮件（若邮箱存在）。
3. 审核拒绝：
   - 填写拒绝理由。
   - 发送拒绝通知邮件（若邮箱存在）。

## 场景6：通联录入（QSO）

1. 进入 `QSL管理 -> 业务 -> 通联记录`。
2. 录入三块信息：
   - 本台配置（日期、时间、时区、模式、设备、天线、功率）
   - 对方配置（呼号、设备、天线、功率、QTH）
   - 信号报告（我方->对方、对方->我方、备注）
3. 提交后生成 QSO 记录；可按规则关联/生成对应卡片记录。

## 场景7：卡片记录维护

1. 进入 `QSL管理 -> 业务 -> 卡片记录`。
2. 可查看/筛选卡片类型（QSO/SWL/EYEBALL）与状态。
3. 按权限执行编辑（操作员及以上），维护卡片日期时间、状态、关联地址等。

## 场景8：打印与发信确认

1. 通过导出接口或打印桥接程序拉取待打印数据（卡片/信封）。
2. 打印完成后回写：
   - 卡片已打印（`productionStatus=PRINTED` 等）
   - 信封已打印（`envelopePrinted=true`）
3. 进入 `QSL管理 -> 业务 -> 发信确认`：
   - 默认筛选待发卡
   - 勾选记录后确认发信
4. 发信后前台查询可见“本台已发卡”，并触发发信通知邮件（邮箱存在时）。

## 场景9：收信确认（对方签收）

1. 对方通过二维码访问收信确认页面（可带参数自动填充），卡片路由：`/plugins/qsl-management/widgets/receive-confirm`。
2. 输入/确认呼号并提交收信确认。
3. 卡片状态更新为“已确认收卡”；前台查询同步显示。
4. 系统发送收信确认邮件（邮箱存在时）。

## 场景10：本台收到回卡（收回确认）

1. 进入 `QSL管理 -> 业务 -> 收信确认`。
2. 输入对方呼号搜索：
   - 存在记录：选择记录并确认收信。
   - 不存在记录：新增眼球卡记录后确认收信。
3. 状态更新为“已收回卡”。

## 场景11：补卡流程

1. 在满足补卡条件（如超过配置天数未收）时发起补卡。
2. 系统将卡片/信封状态回退到待打印待发。
3. 重新打印、回写、发信确认。
4. 实际再次发信成功后，补卡次数字段 +1（0 表示非补卡生成）。

## 场景12：地址管理（单点维护）

1. 进入 `QSL管理 -> 数据 -> 地址管理`。
2. 维护呼号对应地址信息（含邮箱）。
3. 业务数据通过地址ID关联，不在各业务重复保存通信地址字段。

## 场景13：导入导出与备份

1. 进入 `QSL管理 -> 数据 -> 导入导出`。
2. 常规导入：上传 CSV/JSON 按规则导入。
3. 备份导出：导出插件范围完整数据。
4. 建议先在测试环境验证再导入生产。

## 场景14：审计与统计

1. 进入 `QSL管理 -> 审计 -> 审计日志`：
   - 按时间、操作人、对象类型、动作、结果筛选
   - 可导出筛选结果 CSV
2. 进入 `QSL管理 -> 审计 -> 统计报表`：
   - 在线查看图表与统计数字
   - 前台可查看公开统计（按配置）

## 场景15：前台查询（公开）

1. 页面插入 QSL 查询卡片。
2. 输入呼号查询，显示：
   - 呼号
   - 卡片类型
   - 状态标签（待发/已确认/待收回等）
   - 必要时间信息
3. 不展示隐私地址信息；按系统参数限流。

---

## 推荐走查顺序（端到端）

1. 初始化配置
2. 提交呼号绑定并审核通过
3. 新建 QSO
4. 生成并维护卡片记录
5. 打印回写
6. 发信确认
7. 对方收信确认
8. 本台收回确认
9. 查看前台查询、审计日志、统计报表
